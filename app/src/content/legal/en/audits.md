## 1. Overview

Solaris CET aims for a verifiable security posture:

- public contract address disclosure,
- third-party audits when available,
- responsible disclosure process,
- continuous monitoring and incident response.

## 2. Audit reports

Publish links to:

- audit PDF reports,
- scope and version/commit references,
- remediation notes and timelines.

## 3. Responsible disclosure

Report vulnerabilities responsibly using the Responsible Disclosure page. Do not publish exploits before a fix is available.

## 4. Known limitations

Smart contracts and third-party dependencies can fail. Security is risk management, not a guarantee.
